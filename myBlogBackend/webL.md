## 🏗️ 프로젝트 분석

### 프로젝트 개요
이 프로젝트는 Java로 개발된 블로그 백엔드 시스템으로, 사용자들이 게시물을 작성하고 관리할 수 있는 기능을 제공합니다. 주요 기능으로는 게시물의 생성, 수정, 삭제 및 카테고리별, 사용자별 게시물 조회 등이 포함됩니다.

### 아키텍처
- **사용된 아키텍처 패턴**: MVC (Model-View-Controller)
- **주요 레이어/모듈과 역할**:
  - **Controller**: 사용자 요청을 처리하고 적절한 서비스 호출을 담당합니다. 예를 들어, `PostsController`는 게시물 관련 요청을 처리합니다.
  - **Service**: 비즈니스 로직을 처리하며, 데이터베이스와의 상호작용을 관리합니다. `PostService`가 그 예입니다.
  - **Mapper**: 데이터베이스와 객체 간의 변환을 담당합니다. `PostMapper`가 데이터베이스 엔티티와 도메인 객체 간의 변환을 수행합니다.
- **프론트엔드/백엔드 구조**: 이 프로젝트는 백엔드 시스템으로, 프론트엔드와의 통신은 RESTful API를 통해 이루어집니다.

### 핵심 워크플로우
1. **요청**: 클라이언트가 HTTP 요청을 보냅니다.
2. **라우터**: 요청이 적절한 컨트롤러로 라우팅됩니다.
3. **컨트롤러**: 요청을 처리하고 필요한 데이터를 서비스 레이어에 요청합니다.
4. **서비스**: 비즈니스 로직을 처리하고 데이터베이스와 상호작용합니다.
5. **DB**: 데이터베이스에서 데이터를 조회하거나 변경합니다.
6. **응답**: 처리 결과를 클라이언트에 반환합니다.

### 주요 컴포넌트
- **PostsController.java**: 게시물 관련 요청을 처리하는 컨트롤러입니다.
- **PostService.java**: 게시물 관련 비즈니스 로직을 처리합니다.
- **PostMapper.java**: 데이터베이스와 객체 간의 변환을 담당합니다.
- **application.properties**: 애플리케이션 설정 파일입니다.
- **build.gradle**: 프로젝트의 빌드 설정을 관리합니다.

### 주요 기능
- **게시물 조회**: 특정 게시물, 카테고리별 게시물, 사용자별 최근 게시물 조회
- **게시물 생성/수정/삭제**: 게시물의 CRUD 기능 제공
- **기타 기능**: 인기 게시물 조회, 공개 게시물 조회
